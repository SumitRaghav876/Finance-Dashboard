Docs › Authentication
Authentication
Register or login to receive a JWT token. Include it as a Bearer token in the Authorization header for all protected endpoints.
POST /auth/register Create a new user account public ▼
Request body
{ "name": "Sumit Raghav", "email": "sumit@example.com", "password": "password123", "role": "admin|analyst|viewer" }
Response 201
{ "success": true, "data": { "token": "<jwt_token>", "user": { ... } } }
POST /auth/login Login and receive a token public ▼
Request body
{ "email": "sumit@example.com", "password": "password123" }
Response 200
{ "success": true, "data": { "token": "<jwt_token>", "user": { ... } } }
Docs › Transactions
Transactions
Create and manage financial records. Deletions are soft — records are flagged with isDeleted and preserved for audit purposes.
GET /transactions List all transactions auth required ▼
Allowed roles
adminanalystviewer
GET /transactions/:id Get a single transaction auth required ▼
Allowed roles
adminanalystviewer
POST /transactions Create a new transaction auth required ▼
Allowed roles
admin
Request body
{ "amount": 5000, "type": "income | expense", "category": "salary | food | ...", "date": "2024-06-01", "description": "optional" }
Response 201
{ "success": true, "data": { "transaction": { "_id": "...", "amount": 5000, ... } } }
PATCH /transactions/:id Update a transaction auth required ▼
Allowed roles
admin
Request body — any updatable fields
{ "amount": 6000, "description": "Updated description" }
DELETE /transactions/:id Soft delete a transaction auth required ▼
Allowed roles
admin
Sets isDeleted: true on the record. The transaction is preserved in MongoDB for audit and compliance purposes and automatically excluded from all queries.
Docs › Dashboard
Dashboard analytics
Aggregation-powered endpoints for real-time financial insights. All data computed server-side using MongoDB aggregation pipelines.
GET /dashboard/totals Income, expenses & net balance auth required ▼
Allowed roles
adminanalyst
Response 200
{ "totalIncome": 15000, "totalExpense": 6500, "netBalance": 8500 }
GET /dashboard/categories Spending breakdown by category auth required ▼
Allowed roles
adminanalyst
Response 200
[ { "_id": "salary", "total": 10000 }, { "_id": "food", "total": 2500 } ]
GET /dashboard/trends/monthly Monthly income vs expense trends auth required ▼
Allowed roles
adminanalyst
Response 200
[ { "_id": { "month": 6, "year": 2024 }, "total": 5000 } ]
GET /dashboard/activity Recent transaction activity auth required ▼
Allowed roles
adminanalystviewer
Returns the 10 most recent transactions by default. Pass ?limit=N to customize.

Docs › Error codes
Error codes
All errors return a consistent JSON structure. Use the status code and message to handle errors in your client.
Error response shape
{ "success": false, "message": "Descriptive error message" }
Status Meaning Common cause
200 OK Request succeeded
201 Created Resource created successfully
400 Bad request Validation failed — check your request body
401 Unauthorized Missing, invalid, or expired JWT token
403 Forbidden Your role doesn't have access to this endpoint
404 Not found Resource doesn't exist or was soft deleted
500 Server error Unexpected server-side error
